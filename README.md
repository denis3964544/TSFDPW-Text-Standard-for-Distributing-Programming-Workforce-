TSFDPW (Text Standard for Distributing Programming Workforce)

стандарт для написания расписания программирования TSFDPW

правила rules

"6 правил DenSystem

1. Разделение блоков пустой строкой

Каждая мысль или кусок информации отделяется пустой строкой. Это главный визуальный разделитель.

2. Свободный формат записи

Можно писать как угодно - строго структурировано, хаотично или смешанно. DenSystem остаётся системой, потому что блоки отделены.

3. Символы и сокращения вместо длинных слов

Используются короткие обозначения (B, T, R, D, З, рас) и формулы. Это превращает текст в "черновик-код".

4. Запрещено использовать кавычки "" для обозначения текста

Если нужно написать название, то только полностью (Tiktok) или сокращённо (T). Кавычки не применяются как формат записи.

5. Повторение сокращений при тексте

Если в блоке используется сокращение (например, T), то при добавлении текста оно должно быть повторено:

TTT text TTT

TTTTTT text TTTTTT

TTT TTT text TTT TTT

Это "золотое правило" повторения.

6. Объявление системы в начале (необязательно)

Можно начать запись с объявления:

Код

Система записания DenSystem

расписание ради распределения нагрузки и продумивания информации

Это поясняет, зачем нужен свиток, но не является обязательным.
